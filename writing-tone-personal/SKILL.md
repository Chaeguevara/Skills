# 사용자 글쓰기 톤 가이드

블로그 https://dfso2222.tistory.com/ 5~10개 글 분석 기반. 한국어 글에만 적용 — 코드 주석, PR 설명, 영어 문서엔 사용하지 않는다.

## When to apply

- 블로그/티스토리 초안 작성·다듬기
- Obsidian 노트의 본문 (`obsidian-knowledge`/`obsidian-daily` 가 호출될 때 본문 톤도 이걸로)
- 시험·코스·책 후기
- 식당·장소·제품 리뷰
- 학습 회고, "이번 프로젝트에서 배운 것" 류 에세이
- 사용자가 명시적으로 "내 톤으로", "블로그 톤으로" 요청할 때

## Core voice (한 줄)

> 반성적·자조적 학습자가 다음 사람을 위해 담담하게 정리해주는 톤.

핵심 4원칙:
1. **친근한 존댓말 기본** — "~합니다 / ~했습니다 / ~인 것 같습니다". 가끔 관찰형 평서("~해 보인다") 섞임
2. **실패와 깨달음을 같이** — 잘한 것만 쓰지 않는다. "원인은 X였습니다" 하고 인정
3. **짧은 문단·명시적 단계** — 한 문단 3~5문장. 절차는 "1단계 / 2단계" 소제목으로
4. **장식 없음** — 이모지·느낌표 남발·과한 감탄 안 씀. 강조는 따옴표나 **굵은글씨**

## Do

- 도입부에 **글을 쓰게 된 계기/맥락 1~2문장**으로 시작 (인삿말 X)
- 본문은 **소제목 + 짧은 문단** 조합. 소제목은 단언형 또는 단계형
- 결론·교훈은 **자기 한계 인정** + **다음 사람을 위한 한 줄 팁** 형태
- 인용·강조는 큰따옴표 `"…"` 또는 **굵은글씨**
- 숫자가 들어간 절차는 "1단계", "2단계" 식으로 명시
- 식당/장소 리뷰는 평서형 ("~해 보인다", "~한 느낌이다") 으로 한 발 떨어진 관찰자 톤
- 기술/시험 글은 존댓말 ("~입니다", "~했습니다") 으로 학습자 톤
- 마무리는 **담담하게**: "이상입니다", "참고가 되길 바랍니다", "다음에 시도하실 분께 도움이 되면 좋겠습니다"

## Don't

- 이모지, ㅋㅋ/ㅎㅎ, 느낌표 남발 (글 전체에 ! 0~2개)
- "여러분 안녕하세요!", "오늘은 ~에 대해 알아보겠습니다!" 같은 블로거 정형구
- "결론적으로", "마치며" 같은 AI/교과서 티 나는 마무리
- 무근거 긍정 ("정말 좋았어요!", "강추!")
- 과한 친근함 (반말 전환, "여러분~")
- 광고체·SEO 키워드 stuffing
- "~한 것 같다고 생각합니다" 같은 이중 완화 — 한 번이면 충분

## Patterns (자주 쓰는 어미·표현)

- 어미: "~입니다", "~했습니다", "~인 것 같습니다", "~으로 보인다", "~해 보인다"
- 접속: "그래서", "다만", "결국", "막상"
- 자기 위치 명시: "저자는…", "블로거는…" 식 3인칭 자칭도 가끔 등장
- 강조: **굵은글씨**, "큰따옴표"
- 단계 표시: "1단계 — …", "2단계 — …"
- 회고 구문: "원인은 X였습니다", "X였던 것 같습니다", "X에 가까웠던 것 같습니다"

## Reference quotes (톤 기준점)

실제 블로그에서 가져온 인용. 새 글의 톤이 이 문장들과 같은 결인지로 검증.

> "원본 자료를 제대로 확인하지 않았던 것이 주요 원인입니다."

> "지식 자체보다 지식을 연결하는 그 고리가 더 가까웠던 것 같습니다."

> "부채를 만들면 일단 일하게 되어있다."

> "홍대 스럽지 않다는 표현이 적절해 보인다. 상대적으로 차분한 느낌."

> "완성도 60% 수준의 초안을 만들어 흐름을 유지하면서 직장인이 준비할 수 있다."

## Output checklist

글 초안을 내놓기 전 self-check:

- [ ] 도입부가 인삿말 없이 **계기·맥락**으로 시작하는가
- [ ] 한 문단이 **5문장 이하** 인가
- [ ] **이모지 0개**, 느낌표 2개 이하인가
- [ ] **실패/한계 인정** 문장이 최소 한 줄 있는가 (회고/리뷰류)
- [ ] 마무리가 "결론적으로 ~합니다" 같은 정형구 대신 **다음 사람용 팁**으로 끝나는가
- [ ] 강조가 **굵은글씨/따옴표**로만 되어있는가 (밑줄·이탤릭 X)
- [ ] 같은 어미가 3문장 이상 연달아 반복되지 않는가

## 적용 예시

**Bad (AI 톤):**
> 안녕하세요 여러분! 오늘은 TOGAF 시험 후기에 대해 알아보겠습니다 😊 정말 좋은 경험이었어요!! 결론적으로 강추입니다!

**Good (사용자 톤):**
> TOGAF 시험을 본 지 한 달이 지나서야 후기를 정리합니다. 준비하면서 가장 잘못 짚었던 부분을 먼저 적습니다. **원본 자료를 충분히 정독하지 않은 것**이 주요 원인이었습니다.
>
> 1단계 — 원본 한 달 정독.
> 2단계 — 60% 완성도 초안으로 흐름 잡기.
>
> 다음에 준비하실 분은 돈 주고 요약본부터 사지 마시고, 원본을 천천히 읽는 데 시간을 더 쓰시길 권합니다.
